# Тестовое задание Frontend

## Задача

Нужно разработать небольшое SPA-приложение **“Каталог персонажей”** с получением данных из публичного API.

Приложение должно позволять:

* загрузить список персонажей;
* отобразить карточки персонажей;
* искать персонажей по имени;
* фильтровать по статусу;
* открыть страницу/модальное окно с подробной информацией о персонаже;
* обработать состояния загрузки, ошибки и пустого результата.

В качестве API можно использовать **Rick and Morty API**:

* список персонажей;
* детали персонажа;
* поиск и фильтрация доступны из коробки.


# Обязательные требования

* **TypeScript**

# Функциональные требования

## 1. Главная страница со списком персонажей

На главной странице нужно отобразить список персонажей в виде карточек.

В карточке должны быть:

* имя;
* статус;
* вид;
* изображение.

## 2. Поиск

Добавить поле поиска по имени персонажа.

Требования:

* поиск должен работать по вводу пользователя;
* желательно не отправлять запрос на каждый символ без контроля;
* можно использовать debounce, но это не обязательно.

## 3. Фильтрация

Фильтр по статусу:

* Alive
* Dead
* Unknown

## 4. Детальная информация

При клике на карточку пользователя нужно показать подробную информацию о персонаже.

Формат на выбор:

* отдельная страница;
* модальное окно.

В деталях можно показать:

* имя;
* изображение;
* статус;
* вид;
* пол;
* происхождение;
* последнее известное местоположение.

## 5. Состояния интерфейса

Нужно корректно обработать:

* загрузку данных;
* ошибку запроса;
* пустой список результатов.
