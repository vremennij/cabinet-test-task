# Тестовое задание Backend

# Описание
Ваша задача реализовать микросервис на Go.

Это сервис для приёма и отбора заявок на проекты от внешних инициаторов.

## Участники:

* Внешний пользователь (инициатор проекта) — подаёт заявку без авторизации (или с минимальной).

* Администратор (внутренний сотрудник) — рассматривает заявки и принимает решение.

## Бизнес-процесс:

1. Внешний пользователь:
   1. выбирает тип проекта
   2. заполняет форму (контакты, описание, ожидания)
   3. отправляет заявку

2. Система:
   1. сохраняет заявку в статусе PENDING (на рассмотрении)

3. Администратор:
   1. просматривает заявку
   2. принимает решение: ACCEPTED — принять, REJECTED — отклонить (с указанием причины)

## Жизненный цикл заявки

Переходы между статусами заявки доступны только в соответствии с этим простым графом

```
       (создание)
           ↓
        PENDING
        /     \
       ↓       ↓ 
 ACCEPTED      REJECTED
```

## Роли
| Действие              | Кто может          |
|-----------------------|--------------------|
| Подать заявку         | любой пользователь |
| Получить список типов | любой пользователь |
| Просмотр заявки       | только ADMIN       |
| Принять / отклонить   | только ADMIN       |


## API

API-контранкт сервиса приведён в файле `api.json`.

# Технические требования
1. Сервис должен быть написан на `Go`
2. Для хранения данных должна использоваться любая реляционная база данных (например: Postgres, MySQL и так далее).
    1. Сделайте автоматическое заполнение базы данными, которые необходимы для демонстрации работы приложения: (как минимум два пользователя и два типа проектов)
    2. Тестовые данные должны заполняться автоматически при запуске контейнера с базой или контейнера с приложением 
3. При реализации должен быть использован стандартный HTTP-роутер из `net/http`. 
4. Можно использовать любые дополнительные библиотеки
5. Сервис должен работать на порту `8000`
6. Должны быть реализованы тесты, покрывающие базовую функциональность приложения
7. В репозитории должен быть реализован CI-пайплайн для запуска тестов приложения при push в ветку

# Формат сдачи
1. Прикрепите ссылку на публичный репозиторий на GitHub или GitLab
2. В README укажите логины и пароли тестовых пользователей
3. В репозитории должен содержаться файл `compose.yaml`, который поднимет все контейнеры, необходимые для работы приложения (как минимум сам сервис и базу).
4. При проверке сервис будет запущен командой `docker compose up -d`, запросы будут отправляться на адрес `http://localhost:8000/`
